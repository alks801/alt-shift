# Self-review

Честная самооценка проекта по 10 осям. Не везде 10/10 — стараюсь
калибровать, а не хвалить.

Дата: 2026-05-26.

---

## 1. Понятность кода — 8/10

**+**

- Названные хендлеры (`handleCopy`, `handleAskDelete`) вместо inline-стрелок.
- JSDoc только там, где «почему» неочевидно.
- Type-imports inline.
- `const`-объекты в верху файлов вместо магических литералов.
- Tests читаются как спецификация.

**−**

- Несколько комментариев в CSS чересчур горделивы («`min-width: 0` stops a
  long unbroken word…» — можно короче).
- `useCopyToClipboard.legacyCopy` — древний `document.execCommand` fallback,
  который в 2026 уже не нужен (а `crypto.randomUUID`-fallback мы снесли).
  Непоследовательно.

## 2. Масштабируемость кода — 6/10

**+**

- Слои `ui → layout → letters`.
- Versioned envelope в storage.
- AI-клиент изолирован.
- Tokens централизованы.
- Миграции схемы — понятный путь (bump ключа + one-shot migration).

**−**

- `LettersContext` — один большой контекст, все consumers ререндерятся на
  любой мутации. Норм до ~100 писем, дальше нужен `use-context-selector`
  или сплит на чтение/мутации.
- Нет виртуализации (упрётся в ~500 карточек).
- Нет i18n (все строки в EN захардкожены).
- Нет ErrorBoundary.
- `localStorage` quota обрабатывается silent swallow — пользователь может
  потерять данные и не узнать.

## 3. Понимание дизайна — 7/10

**+**

- Каждое spacing/font-size/color пришло из Figma.
- Названия токенов совпадают с языком дизайна.
- Поймал нюансы (240px фикс, soft 1200 как стиль, tabular-nums на
  прогрессе).

**−**

- Первая итерация была с `lucide-react`, хотя бренд-SVG ждали — потеряли
  2 раунда.
- `GoalBanner` CTA имел другие размеры чем «Create New» на дашборде —
  заметил только когда ткнули.
- Пока не было ассетов — нафантазировал Sparkles в EmptyState.

## 4. Внимание к деталям — 8/10

**+**

- Tabular numerals на счётчике (`1 → 2 → 3` не дёргается).
- `scrollbar-gutter: stable` после первого jump при открытии диалога.
- `preview text` коллапсит `\n` в пробелы для карточки.
- `prefers-reduced-motion` уважается везде.
- `min-width: 0` на grid-children для overflow.
- `aria-live="polite"` на превью.

**−**

- Несколько раундов на починку layout (flex-basis vs padding на `/new`) —
  должно было кликнуть с первого раза, а не с третьего.
- Баг с белым фоном `check.svg` оказался кешем браузера, но я
  диагностировал не сразу.

## 5. Качество дизайн-системы — 7/10

**+**

- Токены — единственный источник правды.
- Композабельные примитивы (`Button` → `buttonClassName()` → `ButtonLink`
  без `<a><button>`).
- `Title` с size-вариантами вместо N компонентов.
- `IconAction` как паттерн «текст+иконка inline».
- Слои не протекают: `ui/` ничего не знает про `letters`.

**−**

- Переусложнял в первую итерацию (`Text`, `IconButton`, `Button.ghost` /
  `danger` / `sm`) — снёс в KISS-pass, но итерации стоили времени.
- `Title.{sm,md,lg}` опирается на глобальную type-scale
  (`--font-size-2xl/4xl/display`) — у дизайн-системы зрелого продукта
  была бы отдельная heading-scale.
- Нет Storybook / визуального каталога — примитивы тестируются через
  usage, не в изоляции.
- Нет motion-токенов сверх двух transition-duration.

## 6. Качество вёрстки и адаптивности — 6/10

**+**

- Два консолидированных брейкпоинта, задокументированы в `tokens.css`.
- Auto-scroll к preview на stacked layout.
- `grid-template-columns: minmax(0, 1fr) minmax(0, 1fr)` решает проблему
  overflow корректно.
- CTA full-width на мобиле — норм паттерн.

**−**

- **Не проверял мобильную вёрстку на реальных экранах** — у меня нет
  DevTools, в чате это явно сказал. Это реальный риск, надо прогнать
  вручную.
- Карточки 240px фиксом на мобиле — твой выбор по спецификации, но на
  360px с длинным текстом fade-маска агрессивная.
- Tap-таргеты `iconLink` 40×40 ниже Apple HIG (44) — твоё дизайнерское
  решение, но a11y-аудит может прицепиться.
- Tablet portrait (768px) попадает в мобильный бакет — на больших
  телефонах ещё ок, на планшете в landscape ощущения «пустого экрана».

## 7. React + best practices — 7/10

**+**

- `"use client"` только где нужно.
- `useRef` для AbortController, нет лишнего state.
- `forwardRef` на Field-примитивах.
- AbortController + timer cleanup в effect.
- Tests через Testing Library без снапшотов.
- Нет `useEffect` для derived state.
- Server-route на границе, не размазан по компонентам.

**−**

- `LetterPreview` обёрнут в div с CSS-хаком для grid-stretch — следовало
  бы пробросить ref через `forwardRef`, было бы чище (React 19 даже
  позволяет `ref` как обычный prop).
- `LettersContext` без селектора (см. п.2).
- `LetterForm` на каждое keystroke создаёт новый объект → каскад
  ререндеров (на этом масштабе ОК, но фактический подход — `React.memo` +
  `useCallback` на onChange).
- Нет Suspense boundary вокруг асинхронного контента.
- Нет оптимистичного UI на удаление (хотя там и сейчас быстро).

## 8. Структура проекта — 8/10

**+**

- Чёткие слои.
- Co-located CSS-модули.
- `lib/` для не-React, `components/` для React.
- `tests/` зеркалит структуру.
- Barrel `index.ts` — чистые импорты.

**−**

- `lib/letters/LettersContext.tsx` — React-специфичный, но лежит в `lib/`.
  Корректнее было бы в `components/providers/` или `app/providers/`.
- `lib/hooks/useCopyToClipboard` — то же.
- `tests/` плоский — фрагментарно ок, при 30+ файлах станет неудобно.
- Нет общего `types/` (типы co-located, для маленького проекта норм, при
  росте — затыкания).

## 9. Проработка edge-cases — 7/10

**+**

- Битый JSON / wrong version в storage → `[]`.
- `clipboard.writeText` ошибка → `false`.
- `AbortController` на генерации.
- Hydration race в `useLetters` (поймал и закрыл тестом).
- Пустая trimmed-генерация → возврат в `idle`.
- Cap на goal=5 в обе стороны.
- Banner показывается только после successful generation в этой сессии,
  а не просто потому что есть письма.

**−**

- Нет client-side timeout на генерацию — если OpenAI зависнет,
  пользователь ждёт вечно.
- Нет retry на 429/5xx.
- Нет «уверены?» при уходе с `/new` с введёнными данными — пропадут.
- Нет offline-detection.
- `localStorage` quota exceeded → silent — данные пропали тихо.
- Нет защиты от double-submit на молниеносных кликах до propagation
  `disabled`.

## 10. Продуктовые решения — 7/10

**+**

- «Try Again» overwrites, а не appends — защита от флуда дашборда
  черновиками.
- Banner только после успеха — не мотивирует-стыдит того, кто ещё ничего
  не сделал.
- Floating orb вместо стриминга текста — спокойнее.
- Soft-limit вместо hard-cap на textarea — дружелюбнее.
- Auto-scroll к preview на мобиле — фиксит «куда оно делось?».
- `Copy` disabled-but-visible на idle — discoverability.

**−**

- Delete без undo (toast с undo был бы классичнее).
- Нет концепции draft — закрыл `/new` с заполненными полями → всё
  потеряно.
- Из дашборда нельзя посмотреть полное письмо (карточка clamped, в Future
  work).
- Нет shortcut Cmd+Enter на Generate.
- Empty-state копи упоминает «5 applications» когда count=0 —
  холодновато, можно теплее.

## Приоритет фикса

Если бы это было ревью реального продукта, а не тестового:

1. **Мобильная вёрстка не прогнана руками** (п.6) — единственный реальный
   gap «без alibi». Открыть Responsive mode и пройтись по `/`, `/new`,
   диалогу удаления на 360/390/414/768/960.
2. **LettersContext без селектора** (п.2, п.7) — поломается раньше других
   вещей при росте.
3. **`localStorage` quota silent swallow** (п.9) — пользователь теряет
   данные молча. Минимально нужен toast «Не удалось сохранить» + кнопка
   retry.
4. **Нет timeout на генерацию** (п.9) — приходит реально, не теоретически.
5. **Драфт-режим на `/new`** (п.10) — частая жалоба у форм с длинным
   вводом.

Остальное — справедливо для масштаба тестового задания.
