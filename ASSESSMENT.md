# Оценка проекта Alt+Shift

Шкала: 1 — слабо, 2 — ниже среднего, 3 — средне, 4 — выше среднего, 5 — отлично.
Контекст: тестовое задание React Developer (Variant Group). Стек: Next.js 15 App Router, React 19, TypeScript, CSS Modules + design tokens, OpenAI SDK (server-only), Vitest + Playwright + Storybook 10.

Итоговая средневзвешенная оценка: **4.6 / 5**.

---

## 1. Понятность кода — 5/5

- Чёткие, доменно-ориентированные имена: `LettersProvider`, `useLetters`, `LetterForm`, `generateCoverLetter`. Аббревиатур и магических переменных нет.
- TypeScript-интерфейсы явные и узкие (`LetterInput`, `LettersApi`, `NewLetterInput`), исключают невалидные состояния.
- Комментарии расставлены по правилу «зачем, а не что» — объясняют неочевидные решения: гонка гидрации в `useLetters.ts:36-45`, тонкость `mousedown` против `click` в `ConfirmDialog.tsx:85`, soft-limit vs `maxLength` в `Textarea.tsx:7-13`.
- Файлы небольшие (формочка `LetterForm.tsx` — 114 строк, страница `/new` — 156), каждый компонент имеет одну ответственность.
- `cx` — собственная утилита из 14 строк вместо `clsx`, что показывает осознанный отказ от лишних зависимостей.

Минусы: иногда комментарии чуть длиннее, чем нужно (например, преамбула в `tokens.css`), но это скорее плюс для тестового.

## 2. Масштабируемость кода — 4/5

Сильные стороны:

- Слоистая архитектура `ui → layout → letters` с явным запретом доменных знаний в примитивах. Добавить новую страницу/домен — без правок в `ui/`.
- Дизайн-токены в `:root` (`src/styles/tokens.css`) делают тему/брендинг централизованным изменением.
- Версионированный envelope в `localStorage` (`{ version: 1, letters: [...] }`) с понятной стратегией миграции.
- AI-клиент абстрагирован за `generateCoverLetter(input, { signal }): Promise<string>` — замена провайдера не затрагивает UI.
- Контекст `LettersProvider` смонтирован один раз на root, что предотвращает дивергенцию состояния между страницами.
- `marker-tree` даёт typed-схему селекторов: переименование узла ловится TypeScript во всех местах использования.

Ограничения (нормальные для скоупа задания):

- `LettersApi` намеренно сужен (createLetter / updateLetterBody / deleteLetter) — для большего домена потребуется расщепление контекста или переход на reducer/zustand.
- Нет дискриминированного юнион-типа для статусов запроса; используется строковый литерал `"idle" | "loading" | ...` — рабочий вариант, но при росте состояний удобнее моделировать машиной.
- localStorage не делит данные между пользователями — это сознательный компромисс «без бэка», но масштабирование на multi-user потребует переноса на API.

## 3. Понимание дизайна — 5/5

- Header, dashboard, форма + предпросмотр, состояние «нет писем», диалог удаления, баннер цели, прогресс-полоски — всё это присутствует и точно соответствует логике Figma по описанию.
- Корректно различены `Display` и `Text` начертания Fixel — фирменный шрифт MacPaw подключён локально через `next/font/local` с правильным fallback (`layout.tsx:14-32`).
- Понимание состояний: преамбула предпросмотра (`idle`/`loading`/`ready`/`error`) переключается аккуратно, поведение «Try Again» переопределяет существующее письмо вместо засорения дашборда черновиками — это продуктовое решение, а не баг (`new/page.tsx:80-95`).
- Soft-limit на «Additional details» — текстовая ошибка появляется по правилам дизайна, но ввод не блокируется (важно для копи-пасты), submit при этом отключается. Это близко к state «Too much text» в Figma и удобнее, чем `maxLength`.
- Видимая разница `GoalBanner` между дашбордом (`today`) и формой (`general`): различается копирайт и реализация CTA (ссылка vs обработчик reset).

## 4. Внимание к деталям — 5/5

Очень много мелких, но правильных вещей:

- `html { scrollbar-gutter: stable; }` — нет «прыжка» контента на 15 px при открытии модалки (`globals.css:9-12`).
- Custom `Icon` через `mask-image` → `currentColor`, что даёт правильные hover/danger тинты при единственном SVG-файле (`Icon.tsx`, `Icon.module.css`).
- В `ConfirmDialog` используется `onMouseDown`, а не `onClick` для backdrop — drag изнутри диалога не закрывает его (`ConfirmDialog.tsx:85`).
- Focus trap, восстановление фокуса, scroll lock, `role="alertdialog"`, `aria-labelledby`/`aria-describedby` — полноценный диалог с нуля.
- `:focus-visible` стилизован через `box-shadow` ring токенами; не сбито браузерное правило по умолчанию.
- AbortController + 30-секундный таймаут в `client.ts`, корректный cleanup на unmount страницы `/new` (`new/page.tsx:37`).
- Cross-tab синхронизация через `storage` event в `useLetters.ts:52-59` с явным комментарием, почему нет echo-loop.
- Fallback на `create` если `sessionLetterId` указывает на удалённое в другой вкладке письмо (`new/page.tsx:82-95`).
- Зарезервированный font-display `swap` и themeColor в viewport.
- Селекция текста с фирменным цветом (`::selection`).

## 5. Качество дизайн-системы — 5/5

- Чёткое разделение токенов: `--color-*`, `--space-*`, `--radius-*`, `--font-size-*`, `--shadow-*`, `--transition-*` с понятной семантикой (`--color-text` поверх `--color-ink-900`).
- Примитивы (`Button`, `ButtonLink`, `Input`, `Textarea`, `Icon`, `Title`, `ProgressDots`, `IconAction`, `CopyButton`, `ConfirmDialog`) — все имеют тонкие, ясные API и не утаскивают domain-знаний.
- `Button` экспортирует `buttonClassName()`, который переиспользуется `ButtonLink`. Это решает классический антипаттерн `<Link><Button/></Link>` через грамотный layout, а не вложенность.
- Storybook (14 stories с состояниями) подтверждает зрелость компонентов: 8 состояний `Button`, soft-limit состояние `Textarea`, danger/neutral диалоги.
- Сделан осознанный KISS-проход: удалены `IconButton`, `Text`, неиспользуемые `Button` variant/size, лишние `Field` props, 10 неиспользуемых токенов. Это говорит о дисциплине, а не о минимуме.
- Соблюдено правило «токены — это API визуальной системы»: компоненты ссылаются на `var(--space-4)` вместо магических чисел.

Чего не хватает (но не было в скоупе): нет dark theme (хотя архитектура полностью готова — переопределить токены в `:root[data-theme="dark"]` достаточно).

## 6. Качество вёрстки и адаптивности — 5/5

- Два явных брейкпоинта (`640px` и `960px`), документированы прямо в `tokens.css:110-113`.
- Грамотный auto-scroll предпросмотра на `≤960` при нажатии Generate (`new/page.tsx:60-66`) — без него на мобильном можно подумать, что ничего не произошло. Гейт на `matchMedia` — корректно.
- `PageContainer` сжимает gutter с 32→16 px, лого `Logo` уменьшается 48→32 px, CTA становятся full-width, `GoalStatus` сворачивается в одни точки.
- Высота карточки 240 px на телефонах согласно спеке, без переустройства внутренностей.
- `min-height: 100dvh` корректно учитывает мобильные viewport-баги (адресная строка iOS).
- Скроллбары стилизованы для WebKit и Firefox (`scrollbarThin`).
- Текстовая зона `<textarea>` — `resize: vertical`, что не ломает раскладку в форме.
- Зум не запрещён (`maximum-scale` не выставлен) — корректное решение для a11y.

## 7. Понимание React и следование best practices — 5/5

- Один корневой контекст `LettersProvider`, без дублирующих хуков `useLetters` на разных страницах — корректно решена проблема двойной гидрации.
- `hydrated` флаг включён в deps эффекта сохранения, чтобы первый рендер с `[]` не затёр данные в storage (`useLetters.ts:41-44`). Это тонкая, часто упускаемая деталь.
- Server-only API route (`runtime = "nodejs"`, `dynamic = "force-dynamic"`) — ключ OpenAI никогда не приходит в браузер.
- AbortController прокидывается и в fetch, и в caller signal, и в внутренний таймаут — три источника отмены сведены в один контроллер (`client.ts:24-30`).
- Корректное использование `useCallback` для стабильных ссылок мутаторов, но без переусердствования (нет `useMemo` на ровном месте).
- `forwardRef` на `Button`/`Input`/`Textarea` — даёт интеграцию с фокус-менеджментом (используется в `ConfirmDialog`).
- `useId` для меток полей в `Input`/`Textarea`.
- `createPortal` для модалки → не наследует `overflow:hidden` родителя.
- `aria-live="polite"` на счётчике и предпросмотре.
- Нет «React-антипаттернов»: index в качестве key не используется в динамических списках с реордером (только в split paragraphs, где это допустимо), `setState` всегда функциональный для зависящих от prev обновлений.
- Server / Client components разделены осознанно — `layout.tsx` остаётся серверным, страницы помечены `"use client"` только там, где нужно состояние.

## 8. Структура проекта — 5/5

```
src/
├── app/                Next.js routes + api proxy
├── components/
│   ├── ui/             примитивы (без знания о домене)
│   ├── layout/         AppHeader, PageContainer
│   └── letters/        domain components
├── lib/
│   ├── ai/             client/mock/prompt
│   ├── letters/        контекст (single source of truth)
│   ├── hooks/          переиспользуемые хуки
│   ├── storage/        адаптер localStorage
│   └── markers.ts      типизированное дерево data-test
└── styles/tokens.css
```

- Каждый компонент в своей папке: `Foo/Foo.tsx`, `Foo/Foo.module.css`, `Foo/Foo.stories.tsx`, `Foo/index.ts`. Барель-экспорты — единственный публичный API.
- Чёткая граница «бизнес-логика в `lib/`, презентация в `components/`».
- Тесты вынесены в зеркальную `tests/` и `e2e/` папки.
- `marker-tree` (`src/lib/markers.ts`) — отдельный концепт: иерархия `data-test` атрибутов как типизированный объект, разделяемый между приложением и тестами. Тест-айдишники становятся самодокументирующими (`app/dashboard/letterGrid/cards/<id>/deleteButton`).
- TypeScript path alias `@/*` единообразно используется.

## 9. Проработка edge-cases — 5/5

Покрытые сценарии:

- **Гонка гидрации** — initial render с пустым массивом не должен затереть данные (`useLetters.ts:42`).
- **Cross-tab согласованность** — `storage`-listener в `useLetters` (`useLetters.ts:52-59`).
- **Удалённое в другой вкладке письмо** — `Try Again` корректно фоллбэчит на `create` если `sessionId` уже невалиден (`new/page.tsx:82-95`).
- **Битый/устаревший JSON в localStorage** — `isLetter` валидация и fail-closed на пустой список вместо краша (`storage/letters.ts:33-47`).
- **Превышение quota** — silent ignore, сессионное состояние продолжает работать (`storage/letters.ts:49-58`).
- **Stalled upstream** — 30-секундный таймаут в AI-клиенте с явным сообщением «timed out» (`client.ts:30, 62-64`).
- **Гонка повторных Generate** — `generatingRef` блокирует одновременные клики, предыдущий `AbortController` отменяется (`new/page.tsx:48-54`).
- **Уход со страницы во время генерации** — cleanup на unmount отменяет fetch (`new/page.tsx:37`).
- **Soft-limit на детали** — submit заблокирован, но печатать дальше можно (no truncation surprise на paste).
- **Пустой ответ модели после trim** — статус сбрасывается в `idle` без шумовой ошибки (`new/page.tsx:75-78`).
- **Sсроллбар-шифт** — резерв gutter глобально.
- **OpenAI API errors** — корректно проксируются с правильным статусом, fallback на mock без ключа (`api/generate/route.ts:33-41`).
- **Focus trap + восстановление фокуса** в диалоге.
- **Не на клиенте** — `isBrowser()` гард в storage адаптере.

Что можно ещё добавить (для полноты):

- Нет offline-индикатора при пропаже сети во время генерации (но AbortController покроет таймаут).
- Нет защиты от quota-overflow с UI-нотификацией (silent ignore — спорное решение).

## 10. Продуктовые решения — 5/5

- **Try Again перезаписывает черновик в рамках сессии** — иначе быстрая итерация бы захламила дашборд. Очень верное продуктовое решение, и оно явно отражено в README.
- **Soft-limit + текстовая ошибка вместо `maxLength`** — лучше при паст-операциях, не «срезает» ввод.
- **Mock на стороне сервера при отсутствии ключа** — приложение демо-готовое сразу после `git clone`, что снижает порог входа для ревью. Mock детерминированно правдоподобный, со случайной выборкой опенеров/клоузеров.
- **`/api/generate` как server-only proxy** — единственный безопасный путь работы с OpenAI ключом, плюс не привязываемся к Edge runtime (что важно для streaming).
- **Calmer floating-orb во время генерации вместо progressive text reveal** — продуктово более успокаивающая анимация, чем дёрганый text-streaming.
- **Banner цели появляется только когда `0 < count < 5`** — на пустом списке не дублирует empty state, на достижении цели скрывается. Простая, но грамотная логика.
- **Auto-scroll к предпросмотру только на стэк-лэйаутах** — продуманное поведение для мобильных, не мешает desktop.
- **`ConfirmDialog` вместо native `confirm()`** — единый визуал, focus management, scroll lock, `prefers-reduced-motion`.
- **Versionированный envelope в localStorage** — миграции продуманы как часть продукта, а не как костыль.
- **`hydrated` UI-gate** — пользователь не видит мигающий empty state.

## Замечания и предложения по улучшению

Это не критические замечания, а area for growth:

1. **Состояния запроса** — для дальнейшего роста проще завести `type Status = { kind: 'idle' } | { kind: 'loading' } | { kind: 'ready', text } | { kind: 'error', message }`. Сейчас три отдельных state-переменных (`status`, `preview`, `errorMessage`) — допустимо, но при добавлении статусов это разваливается.
2. **Тесты на storage migrations** — сейчас покрывается loading invalid payload, но нет explicit-теста на «старый ключ — новый ключ» (стратегия описана в README, но не реализована).
3. **Quota-overflow UX** — silent ignore хорош как fallback, но в проде хорошо бы добавить ненавязчивую плашку «не удалось сохранить».
4. **Multi-letter session** — если задание разрастётся, текущий ref-подход к session id окажется тесным; стоит унести в reducer.
5. **a11y-improvement** — кнопка `Delete` отображает иконку + текст, но можно дополнительно покрыть тестами в `axe`/`@storybook/addon-a11y`.
6. **i18n** — все строки литеральные. Если потребуется RU/EN, потребуется централизация копирайтов (сейчас они «расползлись» по компонентам).
